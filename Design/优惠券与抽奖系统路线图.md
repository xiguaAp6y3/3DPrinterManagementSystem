# 优惠券与抽奖系统路线图

## 1. 文档目标

本文件面向后端和数据库开发，设计阶段2可落地的优惠券与抽奖系统路线。

阶段2目标：

- 管理员可创建优惠券模板（满减、折扣、立减）。
- 管理员可向指定用户发放优惠券（批量或单个）。
- 用户可通过抽奖活动自动获取优惠券。
- 用户下单时可选择使用优惠券，系统自动计算折扣金额。
- 优惠券使用、过期、作废全生命周期可追溯。
- 抽奖活动可配置奖品概率、参与次数限制，中奖结果不可篡改。

本系统与阶段1已有订单、报价、收款体系集成，不接在线支付，折扣金额在人工收款确认前生效。

## 2. 核心概念与边界

### 2.1 优惠券模板与用户优惠券的关系

采用「模板 + 实例」两层模型：

- `coupon_templates`：优惠券模板，管理员定义券的类型、面额、规则、有效期策略，不直接属于任何用户。
- `user_coupons`：用户优惠券实例，由模板生成，归属某个用户，记录发放来源、使用状态、关联订单。

这样做的原因：

- 同一个模板可被多次发放（管理员批量发放、抽奖多次中奖）。
- 模板修改不影响已发放的实例。
- 实例记录的是发放时刻固化的规则快照，避免模板后续调整导致已发券金额变化。

### 2.2 发放来源

`user_coupons.source` 标识发放渠道：

- `admin_grant`：管理员手动发放。
- `lottery`：抽奖中奖自动发放。
- `signup_gift`：注册赠券（后续扩展，阶段2预留）。
- `promotion`：活动批量发放（后续扩展，阶段2预留）。

### 2.3 优惠券类型

`coupon_templates.discount_type`：

- `fixed`：固定金额立减，例如满 100 减 20。
- `percentage`：百分比折扣，例如打 9 折。
- `fixed_no_threshold`：无门槛立减，例如直接减 5 元。

### 2.4 使用范围

`coupon_templates.scope_type`：

- `all`：全部订单可用。
- `listed_product`：仅上架商品订单可用。
- `custom`：仅定制订单可用。
- `category`：仅指定商品分类可用（需 `scope_category_id`）。
- `product`：仅指定商品可用（需 `scope_product_id`）。

阶段2先实现 `all`、`listed_product`、`custom`，`category` 和 `product` 作为预留。

## 3. 数据库设计

### 3.1 优惠券模板表 coupon_templates

```sql
CREATE SEQUENCE dbo.seq_coupon_no AS BIGINT START WITH 1 INCREMENT BY 1;
GO

CREATE TABLE dbo.coupon_templates (
    id BIGINT IDENTITY(1,1) NOT NULL PRIMARY KEY,
    coupon_no NVARCHAR(50) NOT NULL,
    name NVARCHAR(100) NOT NULL,
    discount_type NVARCHAR(50) NOT NULL,
    discount_value DECIMAL(18,2) NOT NULL,
    min_spend DECIMAL(18,2) NOT NULL CONSTRAINT DF_coupon_templates_min_spend DEFAULT 0,
    max_discount DECIMAL(18,2) NULL,
    scope_type NVARCHAR(50) NOT NULL CONSTRAINT DF_coupon_templates_scope_type DEFAULT N'all',
    scope_category_id BIGINT NULL,
    scope_product_id BIGINT NULL,
    validity_type NVARCHAR(50) NOT NULL,
    valid_days INT NULL,
    fixed_start_at DATETIME2(3) NULL,
    fixed_end_at DATETIME2(3) NULL,
    total_quota BIGINT NULL,
    issued_count BIGINT NOT NULL CONSTRAINT DF_coupon_templates_issued_count DEFAULT 0,
    per_user_limit INT NULL,
    status NVARCHAR(50) NOT NULL CONSTRAINT DF_coupon_templates_status DEFAULT N'active',
    remark NVARCHAR(1000) NULL,
    created_by BIGINT NULL,
    created_at DATETIME2(3) NOT NULL CONSTRAINT DF_coupon_templates_created_at DEFAULT SYSUTCDATETIME(),
    updated_at DATETIME2(3) NOT NULL CONSTRAINT DF_coupon_templates_updated_at DEFAULT SYSUTCDATETIME(),
    row_version ROWVERSION NOT NULL,
    CONSTRAINT UQ_coupon_templates_no UNIQUE (coupon_no),
    CONSTRAINT FK_coupon_templates_category FOREIGN KEY (scope_category_id) REFERENCES dbo.product_categories(id),
    CONSTRAINT FK_coupon_templates_product FOREIGN KEY (scope_product_id) REFERENCES dbo.products(id),
    CONSTRAINT FK_coupon_templates_created_by FOREIGN KEY (created_by) REFERENCES dbo.staff_users(id),
    CONSTRAINT CK_coupon_templates_discount_type CHECK (discount_type IN (N'fixed', N'percentage', N'fixed_no_threshold')),
    CONSTRAINT CK_coupon_templates_scope_type CHECK (scope_type IN (N'all', N'listed_product', N'custom', N'category', N'product')),
    CONSTRAINT CK_coupon_templates_validity_type CHECK (validity_type IN (N'fixed', N'relative')),
    CONSTRAINT CK_coupon_templates_status CHECK (status IN (N'active', N'disabled', N'archived')),
    CONSTRAINT CK_coupon_templates_discount_value CHECK (discount_value >= 0),
    CONSTRAINT CK_coupon_templates_max_discount CHECK (max_discount IS NULL OR max_discount >= 0)
);
GO
```

字段说明：

- `coupon_no`：业务编号，格式 `CP202607090001`，后端基于日期 + sequence 生成。
- `discount_value`：`fixed` 时是立减金额；`percentage` 时是折扣百分比（例如 90 表示 9 折，即减 10%）。
- `min_spend`：最低消费门槛，`fixed_no_threshold` 时应为 0。
- `max_discount`：百分比折扣时的最高减免金额，`NULL` 表示不封顶。
- `validity_type`：`fixed` 表示固定时间段（`fixed_start_at` / `fixed_end_at`）；`relative` 表示自发放之日起 `valid_days` 天有效。
- `total_quota`：发放总量上限，`NULL` 表示不限。
- `issued_count`：已发放数量，发放时原子递增，防止超发。
- `per_user_limit`：每个用户最多持有该模板的券数，`NULL` 表示不限。

```sql
    CONSTRAINT FK_user_coupons_revoked_by FOREIGN KEY (revoked_by) REFERENCES dbo.staff_users(id),
    CONSTRAINT FK_user_coupons_created_by FOREIGN KEY (created_by) REFERENCES dbo.staff_users(id),
    CONSTRAINT CK_user_coupons_source CHECK (source IN (N'admin_grant', N'lottery', N'signup_gift', N'promotion')),
    CONSTRAINT CK_user_coupons_status CHECK (status IN (N'unused', N'used', N'expired', N'revoked')),
    CONSTRAINT CK_user_coupons_time CHECK (valid_until > valid_from),
    CONSTRAINT CK_user_coupons_discount CHECK (discount_amount IS NULL OR discount_amount >= 0)
);
GO

CREATE INDEX IX_user_coupons_user_status ON dbo.user_coupons(user_id, status, valid_until DESC);
CREATE INDEX IX_user_coupons_template ON dbo.user_coupons(template_id);
CREATE INDEX IX_user_coupons_source_ref ON dbo.user_coupons(source, source_ref_id);
GO
```

字段说明：

- `coupon_no`：用户券实例编号，全局唯一，格式 `UC20260709000123`，后端基于 sequence 生成。
- `source_ref_id`：来源关联 ID。`lottery` 时指向 `lottery_records.id`；`admin_grant` 时可指向发放批次记录。
- `status`：`unused` 未使用、`used` 已使用、`expired` 已过期、`revoked` 已作废。
- `valid_from` / `valid_until`：实际有效期，发放时根据模板 `validity_type` 固化计算。
- `discount_amount`：使用时实际抵扣金额，使用后回填，用于审计。
- `expired_at`：被定时任务标记过期的时间。

### 3.3 订单表扩展字段

在 `dbo.orders` 增加优惠券关联和折扣金额字段：

```sql

    coupon_discount_amount DECIMAL(18,2) NOT NULL CONSTRAINT DF_orders_coupon_discount DEFAULT 0;
GO

ALTER TABLE dbo.orders
ADD CONSTRAINT FK_orders_user_coupon FOREIGN KEY (user_coupon_id) REFERENCES dbo.user_coupons(id);
GO

CREATE INDEX IX_orders_user_coupon ON dbo.orders(user_coupon_id);
GO
```

说明：

- `user_coupon_id`：订单使用的用户券 ID，`NULL` 表示未使用优惠券。
- `coupon_discount_amount`：该订单的优惠券抵扣金额。
- 订单实付金额 = `total_amount` - `coupon_discount_amount`，收款以实付金额为准。
- 优惠券状态在订单创建事务内原子置为 `used`，订单取消时回滚为 `unused`。

## 4. 折扣金额计算规则

### 4.1 计算顺序

订单金额计算流程：

```text
1. 计算订单原始总金额 original_amount = SUM(order_items.subtotal)
2. 应用优惠券折扣 coupon_discount
3. 得到实付金额 payable_amount = original_amount - coupon_discount
4. payable_amount 不得小于 0，最小为 0
```

### 4.2 各类型折扣计算

`fixed`（满减）：

```text
if original_amount < min_spend:
    return 409 COUPON_NOT_MEET_MIN_SPEND
coupon_discount = discount_value
if coupon_discount > original_amount:
    coupon_discount = original_amount
```

`fixed_no_threshold`（无门槛立减）：

```text
coupon_discount = discount_value
if coupon_discount > original_amount:
    coupon_discount = original_amount
```

`percentage`（百分比折扣）：

```text
if original_amount < min_spend:
    return 409 COUPON_NOT_MEET_MIN_SPEND
raw_discount = original_amount * (1 - discount_value / 100)
if max_discount IS NOT NULL AND raw_discount > max_discount:
    raw_discount = max_discount
coupon_discount = raw_discount
```

`discount_value` 百分比约定：

- 90 表示 9 折，减免 = 原价 × (1 - 90/100) = 原价 × 10%。
- 不允许大于等于 100，由 CHECK 或后端校验保证。

### 4.3 使用范围校验

下单时校验优惠券范围与订单类型匹配：

```text
if coupon.scope_type == 'all':
    pass
elif coupon.scope_type == 'listed_product' and order.order_type != 'listed_product':
    return 409 COUPON_SCOPE_MISMATCH
elif coupon.scope_type == 'custom' and order.order_type != 'custom':
    return 409 COUPON_SCOPE_MISMATCH
```

### 4.4 使用前置校验

下单选券时必须校验：

- 券状态为 `unused`。
- 当前时间在 `valid_from` 和 `valid_until` 之间。
- 券归属当前用户。
- 满足最低消费门槛。
- 订单类型与使用范围匹配。

## 5. API 设计

### 5.1 管理端优惠券模板

创建模板：

```text
POST /api/v1/admin/coupon-templates
Idempotency-Key: <uuid>

{
  "name": "新人满100减20",
  "discount_type": "fixed",
  "discount_value": 20,
  "min_spend": 100,
  "scope_type": "listed_product",
  "validity_type": "relative",
  "valid_days": 30,
  "total_quota": 1000,
  "per_user_limit": 1
}
```

更新模板：

```text
PATCH /api/v1/admin/coupon-templates/{template_id}
```

规则：已发放过 (`issued_count > 0`) 的模板，`discount_type`、`discount_value`、`min_spend` 不可修改，防止已发券规则被篡改。

查询模板列表：

```text
GET /api/v1/admin/coupon-templates?status=active&page=1&page_size=20
```

### 5.2 管理员发放优惠券

单个发放：

```text
POST /api/v1/admin/coupon-templates/{template_id}/grant
Idempotency-Key: <uuid>

{
  "user_id": 1001,
  "remark": "客服补偿"
}
```

批量发放（指定用户）：

```text
POST /api/v1/admin/coupon-templates/{template_id}/grant-batch
Idempotency-Key: <uuid>

{
  "user_ids": [1001, 1002, 1003],
  "remark": "活动批量发放"
}
```

全部用户发放：

```text
POST /api/v1/admin/coupon-templates/{template_id}/grant-all
Idempotency-Key: <uuid>

{
  "remark": "双11福利"
}
```

规则：

- 发放前校验 `issued_count + 发放数量 <= total_quota`（如有限额）。
- 校验 `per_user_limit`（如该用户已有该模板券数达上限则跳过）。
- 发放事务内原子递增 `coupon_templates.issued_count`，使用 `UPDLOCK` 行锁防并发超发。
- `relative` 有效期在发放时计算 `valid_from = now`、`valid_until = now + valid_days`。
- `fixed` 有效期直接取模板的 `fixed_start_at` / `fixed_end_at`。

## 6. 抽奖核心算法

### 6.1 权重随机选择

```text
func draw(event_id, user_id):
    # 1. 校验活动状态和时间
    if event.status != 'active':
        return 409 LOTTERY_EVENT_NOT_ACTIVE
    if now < event.start_at or now > event.end_at:
        return 409 LOTTERY_EVENT_NOT_IN_TIME

    # 2. 校验用户抽奖次数
    total_drawn = count_records(event_id, user_id)
    if total_drawn >= event.draws_per_user:
        return 409 LOTTERY_DRAWS_EXHAUSTED
    if event.daily_draws_per_user is not None:
        today_drawn = count_records(event_id, user_id, today)
        if today_drawn >= event.daily_draws_per_user:
            return 409 LOTTERY_DAILY_DRAWS_EXHAUSTED

    # 3. 获取启用的奖品及权重
    prizes = get_enabled_prizes(event_id)
    total_weight = sum(p.weight for p in prizes)
    if total_weight <= 0:
        return 500 LOTTERY_NO_PRIZES_CONFIGURED

    # 4. 加权随机
    r = random(0, total_weight)
    cumulative = 0
    selected = None
    for p in prizes:
        cumulative += p.weight
        if r < cumulative:
            selected = p
            break

    # 5. 落库 + 发券
    begin transaction:
        # 幂等校验：同 event_id + user_id + idempotency_key 已存在则返回原结果
        existing = find_by_idempotency(event_id, user_id, idempotency_key)
        if existing:
            return existing.response

        insert lottery_record
        if selected.prize_type == 'coupon':
            coupon = grant_coupon_to_user(user_id, selected.coupon_template_id, source='lottery', source_ref_id=record.id)
            update record.won_coupon_id = coupon.id
        commit
```

### 6.2 防刷策略

- 每用户每活动总次数限制 (`draws_per_user`)。
- 每用户每日次数限制 (`daily_draws_per_user`)。
- 幂等键防连点重复抽奖。
- 可选：基于 IP 的频率限制，在 FastAPI 中间件实现。
- 可选：要求账号实名或注册满 N 天才可抽奖。

### 6.3 随机数安全性

- 阶段2使用 Python `secrets` 模块生成随机数，不使用 `random`，防止可预测。
- 抽奖结果不可篡改：中奖记录一旦写入，`prize_id`、`is_win`、`won_coupon_id` 不允许更新。

## 10. 并发与事务控制

### 10.1 发放防超发

```sql
-- 行锁 + 条件更新，原子递增已发放数
UPDATE dbo.coupon_templates
SET issued_count = issued_count + @grant_count
OUTPUT INSERTED.issued_count
WHERE id = @template_id
  AND (total_quota IS NULL OR issued_count + @grant_count <= total_quota)
  AND row_version = @current_row_version;
-- 若 affected rows = 0，说明超发或并发冲突，回滚
```

### 10.2 抽奖防重复

- 数据库唯一索引 `UX_lottery_records_event_user_idem` 保证同幂等键不可重复。
- 服务层先查询幂等记录，存在则直接返回原响应。

### 10.3 优惠券使用防并发

```sql
SELECT * FROM dbo.user_coupons WITH (UPDLOCK, HOLDLOCK)
WHERE id = @coupon_id;

-- 校验 status = 'unused' 且在有效期内
-- 更新为 used
```

订单创建与优惠券锁定在同一事务，避免券被多订单同时使用。

## 13. 验收标准

阶段2完成后应满足：

- 管理员可创建满减、折扣、无门槛立减三类优惠券模板。

- 抽奖活动可配置奖品权重，中奖概率按权重比例正确分布。
- 同一用户不超过活动抽奖次数限制和每日限制。
- 幂等键防止连点重复抽奖。
- 优惠券和抽奖全链路操作记录在 `operation_logs`。
- 中奖优惠券自动发放到用户账户，立即可在下单时使用。
