# 3D打印农场管理系统路线图

## 1. 项目定位

3D打印农场管理系统面向拥有多台 3D 打印机的工作室、打印农场、定制制造商和小型工厂，提供从客户定制下单、商品购买、模型文件处理、报价、排产、打印机监控、生产执行到发货售后的完整闭环。

系统分为两个主要端，统一使用 Flutter 做多端开发：

- 电脑端管理后台：管理 3D 打印机状态、设备队列、打印任务、订单、模型文件、库存、销售状态、生产进度和运营数据。
- 手机 APP 端：面向客户，支持上架商品浏览购买、个性化定制、表单提交、上传模型、自动/人工报价、订单跟踪和售后沟通。

首期目标不是一次性做完整 MES，而是先完成“客户下单 -> 商家审核/报价 -> 排产 -> 打印执行 -> 状态追踪 -> 发货”的最短业务闭环。

## 2. 推荐技术路线

### 2.1 后端

- Python：后端主语言。
- FastAPI：核心业务 API、设备状态 API、APP 接口、文件上传、报价接口、Webhook。
- Django：管理后台、权限、运营配置、订单人工审核、员工管理。
- SQL Server：核心业务数据库，适合 Windows Server、企业内网和传统制造业场景。
- Redis：设备状态缓存、任务队列缓存、验证码、短期会话。
- Celery / Dramatiq：异步任务，例如模型分析、切片任务、通知推送、报表统计。
- MinIO / S3 对象存储：模型文件、切片文件、商品图片、打印过程截图。

### 2.2 Flutter 多端客户端

电脑端管理后台：

- Flutter Web / Flutter Windows Desktop。
- 重点是高密度生产管理界面，包括设备集中看板、库存中心、销售状态、订单审核、排期日历。
- 桌面端优先适配大屏、表格、筛选器、批量操作和实时状态卡片。

手机 APP 端：

- Flutter Android / iOS。
- 客户端优先实现商品购买、定制表单、模型上传、报价确认、订单跟踪。
- H5 可作为后续补充渠道，优先级低于原生 APP。

Flutter 工程建议：

- 状态管理：Riverpod 或 BLoC。
- 网络请求：Dio。
- 路由：go_router。
- 本地缓存：Hive / SharedPreferences。
- 文件上传：分片上传预留，MVP 可先普通直传到后端。
- 图表：fl_chart 或 Syncfusion Flutter Charts。
- 电脑端表格：使用 DataTable、PaginatedDataTable 或成熟第三方表格组件。

### 2.3 设备接入

不同 3D 打印机生态差异较大，建议分阶段接入：

- 第一阶段：人工录入打印任务状态，先跑通业务。
- 第二阶段：接入 OctoPrint / Klipper Moonraker API，读取打印进度、温度、文件、状态。
- 第三阶段：增加摄像头截图、失败检测、远程暂停/恢复、自动排产。

### 2.4 AI能力预留

可后续接入 AI 辅助：

- 客户定制需求理解。
- 根据文字描述生成定制建议。
- 辅助客服生成报价说明。
- 检查模型可打印性说明。
- 根据历史订单推荐材料、工艺和价格区间。

AI 不应直接决定最终生产价格，最终报价必须由规则引擎或人工审核确认。

## 4. 产品端划分

## 4.1 电脑端管理后台

核心目标：让商家清楚知道“有哪些订单、哪些任务、哪些机器空闲、哪些机器异常、哪些订单快超时”。

主要模块：

- 设备总览。
- 打印机详情。
- 打印任务队列。
- 排期日历。
- 订单管理。
- 销售状态管理。
- 定制需求审核。
- 模型/切片文件管理。
- 报价管理。
- 商品管理。
- 库存中心。
- 生产看板。
- 质检与发货。
- 数据报表。
- 员工权限。

### 4.1.1 设备总览

展示内容：

- 打印机总数。
- 空闲、打印中、暂停、异常、离线数量。
- 设备利用率。
- 今日预计完成任务数。
- 超时任务数。
- 异常设备数。
- 当前打印任务。
- 打印进度。
- 热床温度、喷嘴温度。
- 剩余时间。
- 所用材料。
- 最近异常。

设备状态建议：

- `idle`：空闲。
- `printing`：打印中。
- `paused`：暂停。
- `completed`：已完成待取件。
- `error`：异常。
- `offline`：离线。
- `maintenance`：维护中。

### 4.1.2 打印任务队列

功能：

- 新建任务。
- 从订单生成任务。
- 指定打印机。
- 上传/选择 G-code 文件。
- 设置优先级。
- 调整任务顺序。
- 开始、暂停、恢复、取消。
- 标记失败并关联原因。
- 失败后重新排队。

### 4.1.3 排期管理

功能：

- 按天、周、月查看打印排期。
- 按打印机查看任务占用。
- 按订单交付日期查看紧急程度。
- 支持手动拖拽调整排期。
- 排期时检查打印机状态、材料库存、预计工时和交付日期。
- 支持预留维护时间和非生产时间。

排期状态建议：

- `unscheduled`：未排期。
- `scheduled`：已排期。
- `locked`：已锁定，不允许自动调整。
- `in_progress`：生产中。
- `delayed`：延期。
- `cancelled`：已取消。

### 4.1.4 订单管理

订单类型：

- 标准商品订单。
- 定制订单。
- 模型代打印订单。
- 批量生产订单。

订单状态：

- `pending_quote`：待报价。
- `quoted`：已报价待确认。
- `pending_payment_confirmation`：待收款确认。
- `paid`：已支付。
- `scheduled`：已排产。
- `printing`：打印中。
- `post_processing`：后处理中。
- `quality_check`：质检中。
- `ready_to_ship`：待发货。
- `shipped`：已发货。
- `completed`：已完成。
- `cancelled`：已取消。
- `refunded`：已退款。

### 4.1.5 销售状态管理

销售分为两部分：

- 上架商品销售：商家提前添加可售商品，包含展示图、图集、规格、价格、库存和销售状态。
- 定制销售：客户提交定制表单和模型/参考图片，后台审核、报价、确认后再进入订单和排期。

上架商品销售状态：

- `draft`：草稿。
- `on_sale`：销售中。
- `off_sale`：已下架。
- `sold_out`：售罄。
- `archived`：归档。

销售管理功能：

- 新增和编辑上架商品。
- 上传封面图和展示图集。
- 设置分类、标签、材料、颜色、尺寸、价格。
- 设置 SKU 库存和安全库存。
- 批量上架、下架、售罄。
- 查看商品浏览量、销量、转化率和退款率。
- 查看定制需求转化率、报价通过率和排期等待时间。

### 4.1.6 定制需求审核

功能：

- 查看客户文字需求。
- 查看客户上传图片/模型。
- 标注尺寸、材料、颜色、用途。
- 内部备注。
- 与客户沟通记录。
- 生成报价单。
- 转为正式订单。
- 审核通过后进入报价。
- 报价确认并支付后进入排期。

定制审核状态：

- `submitted`：已提交。
- `reviewing`：审核中。
- `need_more_info`：需补充信息。
- `rejected`：不可制作。
- `quoted`：已报价。
- `quote_confirmed`：报价已确认。
- `scheduled`：已排期。

### 4.1.7 库存中心

管理对象：

- PLA、PETG、ABS、TPU、树脂等材料。
- 品牌、颜色、规格、批次。
- 喷嘴、热床贴、耗材配件等设备备件。
- 上架商品成品库存。
- 在制品库存。
- 库存重量。
- 成品库存数量。
- 安全库存。
- 单价。
- 出入库记录。

库存中心需要与销售、打印任务、排期联动：

- 上架商品下单后锁定成品库存或生成生产任务。
- 定制订单排期前预估并锁定材料库存。
- 打印任务开始后记录预计消耗。
- 打印完成后确认实际消耗。
- 打印失败后记录损耗。
- 低于安全库存时提醒采购。

## 4.2 手机 APP 端

核心目标：让客户可以方便地购买上架商品、提交定制需求、上传模型并追踪订单。

主要模块：

- 首页。
- 商品商城。
- 商品详情。
- 购物车。
- 定制表单。
- 切片文件上传。
- 报价确认。
- 订单列表。
- 订单详情。
- 消息沟通。
- 个人中心。

### 4.2.1 上架商品购买

功能：

- 商品分类。
- 搜索。
- 商品详情。
- 商品展示图和图集。
- 规格选择。
- 材料/颜色选择。
- 销售状态展示：销售中、售罄、已下架。
- 加入购物车。
- 提交订单。
- 等待管理员确认生产与收款。

### 4.2.2 定制下单

客户可填写：

- 切片文件：必填。
- 申请打印的时间：选填。
- 指定某台打印机以及型号：选填。
- 指定耗材颜色：选填。
- 指定耗材类型：选填。
- 是否使用 AMS：必填。
- 该任务需要打印多少盘：必填。

定制订单先进入“待审核”，统一管理员审核通过后由系统生成预计报价，再由管理员确认人工报价和预计排期；客户确认报价并完成线下支付或人工收款确认后，订单进入生产排期。

### 4.2.3 切片文件上传

支持文件：

- `.gcode`
- `.3mf`
- `.bgcode`
- `.zip`

上传后后端需要：

- 存储原始切片文件。
- 生成文件记录。
- 扫描文件大小和类型。
- 记录文件关联的打印参数说明。
- 后续可扩展切片参数解析、预计耗材和预计工时分析。

### 4.2.4 订单跟踪

客户可看到：

- 已提交。
- 审核中。
- 需补充信息。
- 待报价。
- 待收款确认。
- 已排产。
- 打印中。
- 后处理中。
- 质检中。
- 已发货。
- 已完成。

注意：客户侧状态应简化展示，不需要暴露内部打印机编号、详细失败原因和成本信息。

## 5. 核心业务流程

### 5.1 上架商品购买流程

1. 客户进入 APP。
2. 浏览商品。
3. 查看商品展示图、规格、材料、颜色、尺寸、精度和销售状态。
4. 选择材料、颜色、尺寸、精度、数量。
5. 加入购物车。
6. 提交订单。
7. 管理员确认生产要求和收款状态。
8. 系统生成生产排期和打印任务。
9. 排期前锁定材料库存。
10. 生产完成后质检。
11. 阶段1由人工处理发货，系统不接物流。
12. 客户确认收货。

### 5.2 定制订单流程

1. 客户填写定制表单。
2. 客户上传切片文件。
3. 系统生成定制需求单。
4. 统一管理员进行表单完整性审核。
5. 如信息不足，退回客户补充。
6. 生产管理员评估可打印性、材料、时间和排期。
7. 系统生成预计报价和预计交付时间，管理员确认人工报价。
8. 客户确认报价并完成线下支付或人工收款确认。
9. 订单进入排期。
10. 排期系统锁定打印机时间和材料库存。
11. 打印任务分配到设备。
12. 打印完成后进入后处理。
13. 质检通过后发货。

### 5.3 打印机生产流程

1. 订单完成收款确认。
2. 后台创建打印任务。
3. 上传或选择切片文件。
4. 选择打印机或从排期计划生成。
5. 检查并锁定材料库存。
6. 开始打印。
7. 系统集中记录打印机状态、任务状态和材料消耗。
8. 打印完成后标记取件。
9. 进入后处理或质检。
10. 任务完成并回写订单进度与库存流水。

### 5.4 打印失败处理流程

1. 打印机上报异常或人工标记失败。
2. 系统暂停任务。
3. 记录失败原因。
4. 扣减或记录损耗材料。
5. 选择重新打印、改机打印、取消任务。
6. 如影响交期，通知客服处理客户沟通。

## 6. 系统架构

```text
Flutter 多端客户端
手机 APP / 电脑端管理后台
      |
      | HTTPS
      v
FastAPI API ------------------ 对象存储
      |
      v
SQL Server <----------- Django 管理后台
      |
      v
任务队列 Redis/Celery
      |
      v
设备接入服务
      |
      v
OctoPrint / Klipper / 打印机网关
```

建议后端模块：

- `app_api`：客户 APP 接口。
- `admin_api`：后台管理接口。
- `device_service`：打印机状态采集、设备控制。
- `order_service`：订单、报价、支付、售后。
- `production_service`：排产、打印任务、质检、发货。
- `file_service`：模型文件、图片、切片文件。
- `inventory_service`：材料库存、成品库存、库存锁定、库存流水。
- `sales_service`：上架商品销售、定制销售、销售状态、转化统计。
- `notification_service`：短信、APP 推送、站内信。

## 11. 报价规则路线

第一阶段使用人工报价：

- 客户提交需求。
- 后台人工评估。
- 填写价格和预计交期。
- 客户确认后付款。

第二阶段加入半自动报价：

- 按材料单价。
- 按模型重量或体积。
- 按打印时长。
- 按机器小时成本。
- 按后处理费用。
- 按加急系数。

报价公式草案：

```text
总价 = 材料成本 + 机器工时成本 + 人工处理费 + 后处理费 + 包装物流费 + 利润
```

第三阶段加入模型分析：

- 自动读取模型体积。
- 估算材料消耗。
- 估算打印时间。
- 给出报价建议。

## 12. 权限与安全

- 后台员工账号必须按角色授权。
- 客户只能访问自己的订单、文件和报价。
- 模型文件下载需要签名 URL，避免公开泄露。
- 设备远程控制接口必须记录操作日志。
- 支付回调必须校验签名。
- 文件上传需要限制格式、大小和病毒扫描。
- 报价、订单金额、支付金额必须以后端数据库为准。
- 打印机 API 密钥不能暴露给前端。

## 15. 关键风险

- 打印机品牌和固件生态不统一，设备接入要分阶段，不要一开始强依赖自动控制。
- 切片文件包含打印机参数，MVP 阶段先做上传、保存和人工审核，自动解析后置。
- 定制订单很难完全标准化，早期必须保留人工报价和人工确认。
- 上架商品和定制订单必须共用订单体系，但审核、报价、排期逻辑要分开处理。
- 库存必须支持锁定机制，否则收款确认、排期和实际生产之间容易出现材料不足。
- 远程控制打印机有安全风险，需要权限、日志和二次确认。
- APP 端定制表单过长会降低转化，建议分步骤提交。
- 订单状态过细会让客户困惑，客户侧展示应比后台简化。
