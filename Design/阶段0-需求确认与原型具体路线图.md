# 阶段0：需求确认与原型具体路线图

## 1. 阶段目标

阶段0的目标是在正式开发前，把 3D 打印农场管理系统的业务边界、核心流程、页面原型、数据模型和接口边界确认下来。

阶段0不以编码为主，核心交付是：

- 明确 MVP 范围。
- 明确上架商品销售和个性化定制的业务差异。
- 明确库存、打印机状态、销售状态、审核、报价、排期之间的关系。
- 输出 Flutter 电脑端和手机 APP 的核心页面原型。
- 输出 SQL Server 数据库 ER 草案。
- 输出 FastAPI / Django API 草案。
- 输出阶段1开发任务拆分。

## 2. 阶段周期建议

建议周期：7 到 10 个工作日。

如果需求方决策快，可以压缩到 5 个工作日；如果设备接入、报价规则、库存规则复杂，建议保留 2 周。

## 3. 参与角色

- 产品负责人：确认业务范围、页面流程、字段规则。
- 生产负责人：确认打印机状态、排期、失败处理、后处理流程。
- 库存负责人：确认材料库存、成品库存、库存锁定和出入库规则。
- 销售/客服负责人：确认上架商品、定制表单、审核、报价和客户沟通流程。
- 后端负责人：确认 SQL Server 表结构、API 边界、后台权限。
- Flutter 负责人：确认多端页面结构、组件方案和交互边界。
- UI/UX 设计：输出电脑端和手机端原型。

## 4. 阶段0已确认决策

以下问题已确认，可作为阶段1设计和开发输入：

- MVP 同时支持上架商品和个性化定制。
- 上架商品不走现货销售，采用下单后生产。
- 上架商品规格包含材料、颜色、尺寸、精度、数量。
- 商品展示图由后台自行上传，支持封面图、详情图、打印成品图、场景图等类型。
- 个性化定制必须提交切片文件，因为切片文件除模型本身外，还包含打印机参数。
- 定制审核由统一管理员处理。
- 报价采用“系统预计 + 人工确认报价”的方式。
- 报价确认后必须完成支付确认才能排期。
- 阶段1不接在线支付，使用线下支付或人工确认收款作为排期前置条件。
- 排期以订单为单位。
- 一笔订单允许拆成多个打印任务。
- 库存需要区分材料库存、成品库存、在制品库存。
- 打印机状态第一阶段由人工维护，暂不接入 OctoPrint / Klipper。
- 手机 APP 只做客户侧。
- 电脑端只做内部管理。
- 支付和物流不在阶段1上线。

个性化定制表单字段已确认：

- 切片文件：必填。
- 申请打印的时间：选填。
- 指定某台打印机以及型号：选填。
- 指定耗材颜色：选填。
- 指定耗材类型：选填。
- 是否使用 AMS：必填。
- 该任务需要打印多少盘：必填。

## 6. 业务流程设计

### 6.1 上架商品销售流程

阶段0需要确认以下流程：

1. 后台新增上架商品。
2. 上传商品封面图和展示图集。
3. 配置商品规格、材料、颜色、价格。
4. 设置为下单后生产。
5. 设置销售状态：草稿、销售中、售罄、下架、归档。
6. 客户在 APP 浏览商品。
7. 客户选择规格并下单。
8. 系统生成订单。
9. 管理员确认订单和生产要求。
10. 完成线下支付或人工收款确认。
11. 系统生成生产排期和打印任务。

阶段0需要明确的字段：

- 商品名称。
- 商品分类。
- 商品描述。
- 封面图。
- 展示图集。
- 材料。
- 颜色。
- 尺寸。
- 精度。
- 数量规则。
- 价格。
- 销售状态。
- 是否支持定制备注。
- 商品展示图。
- 下单后生产标记。

### 6.2 个性化定制流程

阶段0需要确认以下流程：

1. 客户在 APP 进入定制入口。
2. 客户填写定制表单。
3. 客户上传切片文件。
4. 系统生成定制需求单。
5. 统一管理员审核信息完整性和切片文件。
6. 信息不足时退回客户补充。
7. 生产人员评估可打印性、材料、工时和交期。
8. 系统生成预计报价，管理员确认人工报价。
9. 客户确认报价。
10. 客户完成线下支付或由管理员确认收款。
11. 后台创建订单和打印任务。
12. 生产负责人安排排期。
13. 打印完成后进入后处理、质检、发货。

阶段0需要明确的表单字段：

- 切片文件：必填。
- 申请打印的时间：选填。
- 指定某台打印机以及型号：选填。
- 指定耗材颜色：选填。
- 指定耗材类型：选填。
- 是否使用 AMS：必填。
- 该任务需要打印多少盘：必填。

### 6.3 审核流程

审核状态：

- `submitted`：已提交。
- `reviewing`：审核中。
- `need_more_info`：需补充信息。
- `rejected`：不可制作。
- `quote_pending`：待报价。
- `quoted`：已报价。
- `quote_confirmed`：报价已确认。
- `scheduled`：已排期。

阶段0需要确认：

- 谁可以审核。
- 谁可以驳回。
- 谁可以要求补充信息。
- 谁可以生成报价。
- 审核记录是否对客户可见。
- 客户补充信息后是否回到原审核人。

已确认规则：

- 审核统一由管理员处理。
- 信息不足时退回客户补充。
- 审核不通过时记录原因。
- 报价采用预计报价加人工确认报价。
- 报价确认后必须完成收款确认才能排期。

### 6.4 排期流程

阶段0已确认：

- 排期以订单为单位。
- 一笔订单可以拆成多个打印任务。
- 一个打印任务只能绑定一台打印机。
- 排期时需要检查打印机可用时间、材料库存、预计工时和交付日期。
- 第一阶段支持手动排期，暂不做自动排产。

排期状态：

- `unscheduled`：未排期。
- `scheduled`：已排期。
- `locked`：已锁定。
- `in_progress`：生产中。
- `delayed`：延期。
- `completed`：已完成。
- `cancelled`：已取消。

## 7. 库存与设备集中管理设计

### 7.1 库存中心

阶段0已确认库存分层：

- 材料库存：PLA、PETG、ABS、TPU、树脂等。
- 成品库存：打印完成后已入库或待发货的成品，不作为阶段1上架商品现货销售入口。
- 在制品库存：已开始生产但未完成入库或发货的产品。
- 库存锁定：订单收款确认、排期或生产前占用的材料、成品或在制品。

阶段0输出规则：

- 上架商品下单后如何生成生产需求并锁定材料库存。
- 定制订单排期前如何锁定材料库存。
- 打印失败后如何记录材料损耗。
- 成品入库和出库如何记录。
- 低库存预警阈值如何配置。

### 7.2 打印机状态中心

阶段0已确认打印机状态由人工维护，状态枚举如下：

- `idle`：空闲。
- `printing`：打印中。
- `paused`：暂停。
- `completed`：已完成待取件。
- `error`：异常。
- `offline`：离线。
- `maintenance`：维护中。

阶段0需要设计的设备字段：

- 打印机名称。
- 品牌。
- 型号。
- 打印类型。
- 可用材料。
- 打印尺寸。
- 所在位置。
- 当前状态。
- 当前任务。
- 是否支持设备 API，阶段1仅预留字段。
- 设备备注。

阶段1实施规则：

- 先做人工维护设备状态。
- 保留 OctoPrint / Klipper 接入字段。
- 不在阶段1强制实现远程控制。

## 8. Flutter 原型设计路线

### 8.1 电脑端管理后台原型

必须设计的页面：

- 登录页。
- 生产总览。
- 打印机集中看板。
- 打印机详情。
- 打印任务队列。
- 排期日历。
- 销售状态管理。
- 上架商品管理。
- 商品展示图管理。
- 定制需求审核。
- 报价管理。
- 订单管理。
- 库存中心。
- 材料库存。
- 成品库存。
- 质检与发货。

重点原型要求：

- 打印机状态必须一眼可见。
- 异常设备、延期任务、低库存必须有醒目标识。
- 表格必须支持筛选、搜索、状态过滤。
- 商品管理必须能看到展示图和销售状态。
- 定制审核页必须能看到客户表单、文件、审核记录、报价入口。
- 排期页必须能按打印机和日期查看任务。

### 8.2 手机 APP 原型

必须设计的页面：

- 首页。
- 上架商品列表。
- 上架商品详情。
- 商品图集预览。
- 购物车。
- 订单提交。
- 定制入口。
- 定制表单。
- 文件上传。
- 定制需求详情。
- 报价确认。
- 我的订单。
- 订单详情。
- 个人中心。

重点原型要求：

- 上架商品详情必须突出展示图、规格、价格和销售状态，并说明下单后生产。
- 定制表单必须分步骤，避免一次性长表单。
- 文件上传后必须显示上传状态。
- 报价确认必须展示价格、交期、说明和确认按钮。
- 订单详情必须用客户可理解的状态，不展示内部复杂状态。

## 11. 技术验证任务

阶段0建议做轻量技术验证，不进入完整开发。

### 11.1 Flutter 多端验证

验证项：

- Flutter Android 能正常调用本地或测试 API。
- Flutter Web / Windows 能跑通管理后台基础布局。
- 同一套设计语言能覆盖移动端和电脑端。
- 文件上传组件在手机端可用。
- 管理后台表格、筛选、状态卡片可实现。

### 11.2 SQL Server 验证

验证项：

- Python 后端可连接 SQL Server。
- Django 可使用 SQL Server 驱动。
- FastAPI 可通过 SQLAlchemy / pyodbc 访问 SQL Server。
- 中文字段、金额字段、时间字段存取正常。

### 11.3 文件上传验证

验证项：

- 支持图片上传。
- 支持 `.stl`、`.obj`、`.3mf`、`.step`、`.stp`、`.zip` 文件上传。
- 文件大小限制策略明确。
- 文件存储路径和访问权限明确。
